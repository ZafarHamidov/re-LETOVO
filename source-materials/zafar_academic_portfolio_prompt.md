# Prompt-инструкция для Codex / Claude Code
## Проект: персональная академическая страница Зафара на основе файлов из текущей директории

Ты работаешь в среде **Codex** или **Claude Code** как:
- admissions strategist
- academic portfolio architect
- data extraction analyst
- frontend/web builder
- GitHub Pages deployment assistant

Твоя задача: на основе файлов, которые уже лежат в **текущей директории** и вложенных папках, создать **персональную академическую страницу Зафара** в формате сильного, аккуратного и убедительного **portfolio website / academic dashboard**.

Эта страница должна одновременно выполнять 4 функции:

1. **Витрина достижений**  
   Показывать, чего Зафар уже добился.

2. **Трек поступления**  
   Показывать цели, дорожную карту и текущую готовность к поступлению в топовые вузы.

3. **Система контроля прогресса**  
   Показывать, что уже сделано, что в работе, чего еще не хватает.

4. **Публичная презентабельная ссылка**  
   Чтобы Зафар мог при необходимости отправить страницу преподавателю, ментору, консультанту, родственнику, знакомому, interview panel или admissions helper и показать свой уровень.
   
посмотри что еще можно добавить или предложить для улучшения.
---

## Ключевой результат

Сделай сайт, который выглядит как личный академический кабинет сильного школьника с амбициями поступления в топовые университеты.

На странице должны быть отражены:

- кто такой Зафар
- его учебный профиль
- его сильные стороны
- его достижения
- его успеваемость
- его академический и внеакадемический трек
- его цели по поступлению
- его текущий уровень готовности
- пробелы и то, что нужно добрать
- ссылки на дипломы, сертификаты, отчеты, PDF и иные подтверждения

---

## Где брать данные

Используй **только локальные файлы из текущей директории** и вложенных подпапок.

Нужно автоматически найти и проанализировать:

- `*.pdf`
- `*.html`
- `*.mhtml`
- `*.jpg`
- `*.jpeg`
- `*.png`

Особенно внимательно обрабатывай файлы, в названии которых встречаются слова вроде:

- успеваемость
- report
- репорт
- оценки
- grades
- олимпиада
- олимпиада2
- достижения
- достижения / activities
- диплом
- дипломы
- сертификат
- пробные экзамены
- exams
- half-term
- school report
- Letovo / Летово
- activities
- личный кабинет

Если есть изображения-скриншоты, используй их как дополнительные доказательства, но основные факты лучше брать из PDF / HTML / MHTML.

## Что нужно сделать по шагам

### Шаг 1. Просканировать директорию
1. Рекурсивно обойди текущую директорию.
2. Построй список релевантных файлов.
3. Сгруппируй их по категориям:
   - academic performance
   - olympiads / competitions
   - certificates / diplomas
   - activities / extracurriculars
   - diagnostics / mock exams
   - teacher / school reports
   - admissions-related evidence

### Шаг 2. Извлечь содержимое
Для каждого файла:
- извлеки текст
- определи тип документа
- выдели ключевые факты
- сохрани их в структурированном JSON

Пример целевой структуры данных:

```json
{
  "student": {
    "name": "Zafar",
    "school": "Letovo",
    "grade_level": "",
    "academic_years": []
  },
  "academics": [
    {
      "source_file": "",
      "period": "",
      "subject": "",
      "grade": "",
      "teacher_comment": "",
      "evidence_link": ""
    }
  ],
  "olympiads": [
    {
      "title": "",
      "level": "",
      "result": "",
      "date": "",
      "evidence_link": ""
    }
  ],
  "activities": [
    {
      "name": "",
      "role": "",
      "period": "",
      "impact": "",
      "evidence_link": ""
    }
  ],
  "certificates": [
    {
      "title": "",
      "issuer": "",
      "date": "",
      "evidence_link": ""
    }
  ],
  "mock_exams": [
    {
      "exam": "",
      "date": "",
      "score": "",
      "max_score": "",
      "percent": "",
      "evidence_link": ""
    }
  ],
  "goals": [
    {
      "title": "",
      "deadline": "",
      "status": "planned"
    }
  ],
  "admissions_gap_analysis": {
    "strengths": [],
    "weaknesses": [],
    "missing_items": [],
    "next_actions": []
  }
}
```

### Шаг 3. Нормализовать факты
При извлечении:
- убирай дубли
- помечай сомнительные данные как `needs_review`
- не выдумывай недостающие факты
- если данные противоречат друг другу, показывай это явно
- у каждого важного тезиса должна быть привязка к исходному файлу

### Шаг 4. Построить профиль Зафара
На основе извлеченных данных собери профиль по блокам:

#### 1. Hero section
Коротко и сильно:
- имя
- школа
- направление интересов
- амбиция: топовые университеты
- 2-3 сильные фразы о его академическом пути

#### 2. Academic Snapshot
Покажи:
- общий учебный профиль
- сильные предметы
- динамику успеваемости
- последние ключевые результаты
- средний академический уровень, если его реально можно честно посчитать

#### 3. Achievements
Покажи:
- олимпиады
- академические награды
- дипломы
- сертификаты
- конкурсы
- исследовательские и иные достижения

#### 4. Extracurriculars
Покажи:
- активности
- проекты
- лидерство
- интеллектуальные интересы
- выступления, кружки, иные серьезные занятия

#### 5. Evidence Vault
Сделай раздел с карточками или списком ссылок на:
- PDF дипломов
- сертификаты
- school reports
- academic evidence
- mock exam reports

#### 6. Admissions Roadmap
Покажи:
- список целевых университетов, если он указан пользователем или найден в локальных заметках
- иначе создай блок-заготовку для ручного заполнения
- текущий статус подготовки
- чего не хватает до сильного профиля

#### 7. Gap Analysis
Отметь:
- что уже есть на хорошем уровне
- чего еще не хватает
- какие элементы профиля желательно усилить

Например:
- standardized tests
- research portfolio
- competitions depth
- leadership depth
- volunteering / impact
- essays / narrative
- recommendations
- public-facing project portfolio
- personal brand / website completeness

#### 8. Progress Tracker
Сделай визуальный трекер:
- completed
- in progress
- planned
- missing

Можно в виде:
- progress bars
- radar chart
- donut charts
- checklist matrix
- timeline

## Обязательные визуальные элементы

Добавь минимум 3 визуальных блока:

### 1. Общий процент готовности
Сделай **Estimated Admissions Readiness Score**, например в процентах.

Важно:
- это должен быть **не “истинный” балл**, а **оценочный внутренний индикатор**
- рядом обязательно покажи методику расчета
- не выдавай это за официальную admissions-оценку

Пример компонентов:
- academics
- olympiads/awards
- extracurricular depth
- testing readiness
- leadership/initiative
- evidence completeness
- narrative readiness
- admissions packaging

### 2. Диаграмма сильных и слабых сторон
Подходит:
- radar chart
- horizontal bar chart
- scorecards

### 3. Timeline / Roadmap
Покажи путь:
- что уже сделано
- что идет сейчас
- что нужно сделать в ближайшие 3, 6, 12 месяцев

---

## Важное правило про честность

Ничего не придумывай.

Если данных не хватает:
- прямо пометь как `missing`
- предложи, что именно нужно добавить вручную
- выведи отдельный блок: **Что еще нужно загрузить / подтвердить**

Если по файлам нельзя надежно вычислить показатель:
- не вычисляй его как факт
- дай аккуратную оценку с пометкой `estimated`
- объясни, на чем она основана

## Требования к сайту

Сделай сайт в формате, подходящем для **GitHub Pages**.

### Предпочтительный стек
Выбери один из вариантов:

#### Вариант A. Самый простой и надежный
- HTML
- CSS
- JavaScript
- JSON с данными

#### Вариант B. Более современный
- Vite
- React
- Tailwind
- статическая сборка для GitHub Pages

Если нет особой причины усложнять, **предпочти Вариант A**.  
Цель: чтобы страница открывалась быстро, была легкой и ее было легко поддерживать.

Сделай отдельный раздел, где можно нажать и открыть:

- дипломы
- сертификаты
- отчеты школы
- полугодовые / годовые оценки

- название
- тип
- дата или период

## Как оценить готовность к топ-вузам

Сделай отдельный аналитический блок:

### Admissions Readiness Framework
Оцени по 10-балльной или 100-балльной шкале:

- Academic strength
- Consistency of grades
- Competition profile
- Intellectual depth
- Extracurricular substance
- Leadership / initiative
- Evidence completeness
- Application readiness
- Testing readiness
- Distinctive profile

Затем:
- покажи сильные стороны
- покажи зоны роста
- покажи недостающие элементы

И выведи:
- **Current profile assessment**
- **What would most increase competitiveness**
- **Top 5 priority actions**

## Техническая инструкция по обработке файлов

### Для PDF
- извлекай текст
- при необходимости доставай таблицы
- если PDF сканированный, применяй OCR только если без этого никак

### Для HTML / MHTML
- достань текст, заголовки, таблицы, списки
- очисти мусор навигации
- выдели именно содержательные академические данные

### Для изображений
- используй как вложения / превью
- OCR только при необходимости

## Инструкция по публикации через GitHub Pages

Если проект статический:
1. Создай GitHub репозиторий
2. Загрузи файлы
3. Включи GitHub Pages
4. Укажи branch:
   - `main`
   - folder: `/root` или `/docs`, в зависимости от структуры
5. Проверь публичную ссылку

Если Vite/React:
- настрой `base` path
- сделай build
- опубликуй папку build/dist в GitHub Pages

- какие документы можно публиковать открыто

- **Public showcase**

## Критерии качества

Готовый результат считается хорошим, только если:

- страница реально собирается и открывается
- есть доказательная база через документы
- видно академические результаты
